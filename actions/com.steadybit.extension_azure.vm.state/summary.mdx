# Introduction

This action leverages Azure API to manipulate the state of Azure virtual machines. More specifically, it can reboot, delete, stop or deallocate them.

# Use Cases

- Verify that system services come up on system start.
- Check that your monitoring solution informs you about this occurrence.

# Rollback

Note that this attack does not support any rollback. The attack only transmits the API call, e.g., Stop or Delete virtual machine.

# Parameters

| Parameter | Description                                                                          | Default |
|-----------|--------------------------------------------------------------------------------------|---------|
| Action    | How the virtual machine state should be changed. Reboot, delete, stop or deallocate. |         |
