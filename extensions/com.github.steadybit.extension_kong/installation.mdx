# Prerequisites

- Kong needs to have the [request-termination plugin](https://docs.konghq.com/hub/kong-inc/request-termination/#example-use-cases) installed (typically installed by default).

 <Accordion title="Helm / Kubernetes">
  To install the chart with the name <code>steadybit-extension-kong</code>.

  <Code lang="bash" code={`
helm repo add steadybit https://steadybit.github.io/helm-charts
helm repo update

helm upgrade steadybit-extension-kong \
  --install \\
  --wait \\
  --timeout 5m0s \\
  --create-namespace \\
  --namespace steadybit-extension \\
  --set kong.name="{{SYMBOLIC_NAME}}" \\
  --set kong.origin="{{KONG_API_SERVER_ORIGIN}}" \\
  steadybit/steadybit-extension-kong`} />

You may optionally define an <a href="https://github.com/steadybit/helm-charts/blob/2c7c40e193fdbe386b10ff08e2547d27d7ac749a/charts/steadybit-extension-kong/values.yaml#L10-L13">authentication header</a> through the `kong.headerKey` and `kong.headerValue` values.

</Accordion>
<Accordion title="Docker">
  You may alternatively start the Docker container manually.

  <Code lang="bash" code={`
docker run \\
  --env STEADYBIT_LOG_LEVEL=info \\
  --env STEADYBIT_EXTENSION_KONG_INSTANCE_0_NAME="{{SYMBOLIC_NAME}}" \\
  --env STEADYBIT_EXTENSION_KONG_INSTANCE_0_ORIGIN="{{KONG_API_SERVER_ORIGIN}}" \\
  --expose 8084 \\
  steadybit/extension-kong:latest
`} />
</Accordion>
